Exemplu BUGS/JAGS si introducere in blocking Gibbs

Aceasta pagina prezinta un exemplu complet de cod BUGS/JAGS pentru modelul ierarhic si introduce conceptul de blocking (gruparea variabilelor) in esantionarea Gibbs.

Cod BUGS pentru modelul ierarhic: Modelul specifica: for(j in 1:J) { w[j] ~ dmnorm(mu_w, Sigma_w.inv); for(i in 1:N[j]) { y[i,j] ~ dnorm(inprod(x[i,j], w[j]), tau) } }. Priorii se adauga separat: mu_w ~ dmnorm(mu0, V0.inv), Sigma_w.inv ~ dwish(S0, eta0).
Blocking Gibbs: In loc de a esantiona fiecare variabila individual, se grupeaza variabilele corelate si se esantioneaza impreuna. De exemplu, esantionarea (mu, sigma^2) simultan in loc de p(mu|sigma^2,...) si apoi p(sigma^2|mu,...) separat.
Motivatia pentru blocking: Cand variabilele sunt puternic corelate a posteriori, Gibbs standard se misca lent (random walk pe directii coordonate). Esantionarea in bloc capteaza corelatia si permite pasi mai mari in spatiul parametrilor.

Comentariu

Rezumat

Pagina face legatura intre teorie si practica prin codul BUGS si introduce blocking ca optimizare a Gibbs. Codul BUGS arata cat de concis poate fi specificat un model ierarhic complex, iar blocking adreseaza problema convergentei lente cauzate de corelatii.

Tipare Cheie

Blocking reduce autocorelatia lantului Markov prin esantionarea comuna a variabilelor corelate. Trade-off-ul este ca blocurile mai mari sunt mai costisitoare de esantionat, dar produc esantioane mai independente. Dimensiunea optima a blocurilor depinde de structura de corelatie.

Aplicatii Practice

In practica, BUGS si JAGS detecteaza automat unele oportunitati de blocking. Stan abordeaza problema diferit prin HMC, care exploateaza geometria distributiei posterioare si nu necesita blocking explicit.

Exemplu din Lumea Reala

In modelele de serie temporala financiara, parametrii de volatilitate si media sunt puternic corelati. Blocking Gibbs care esantioneaza acesti parametri impreuna poate reduce numarul de iteratii necesare cu un ordin de marime fata de Gibbs standard.
