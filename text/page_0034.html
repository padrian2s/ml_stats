Capitolul 1: Invatarea Supervizata - Clasificare

Introducerea invatarii supervizate cu focus pe clasificare, aproximarea de functii si exemplul cu forme colorate.

Clasificare Binara: Clasificare cu C = 2 clase, unde y ∈ {0, 1}.
Clasificare Multi-clasa: Clasificare cu C > 2 clase.
Aproximare de Functii (Function Approximation): Formalizarea clasificarii ca y = f(x), unde f este o functie necunoscuta care trebuie estimata din setul de antrenament.
Generalizare: Capacitatea modelului de a face predictii corecte pe intrari noi, care nu au fost vazute in timpul antrenamentului.
Figura 1.1: (a) Exemple etichetate de forme colorate cu 3 cazuri de test neetichetate (semiluna albastra, cerc galben, sageata albastra). (b) Reprezentarea datelor de antrenament ca o matrice de design N x D, cu caracteristici (culoare, forma, dimensiune) si etichete.
Matricea de Design: Datele de antrenament sunt reprezentate ca o matrice X de dimensiune N x D, unde randul i reprezinta vectorul de caracteristici x_i, iar ultima coloana contine etichetele y_i ∈ {0, 1}.

Comentariu

Rezumat

Sectiunea 1.2 introduce invatarea supervizata prin exemplul clasificarii. Problema este formalizata ca aproximare de functii: se estimeaza o functie f dintr-un set de antrenament etichetat, apoi se fac predictii pe date noi folosind y_hat = f_hat(x). Scopul principal este generalizarea - capacitatea de a performa bine pe date nevazute.

Tipare Cheie

Exemplul cu forme colorate ilustreaza provocarile clasificarii: semiluna albastra este relativ usor de clasificat (toate formele albastre au eticheta 1), dar cercul galben este ambiguu (unele obiecte galbene au eticheta 0, altele 1). Aceasta ambiguitate motiveaza nevoia de predictii probabilistice.

Aplicatii Practice

Matricea de design este reprezentarea standard a datelor in invatarea automata. In practica, caracteristicile pot fi discrete (culoare), continue (dimensiune) sau combinatii ale acestora. Generalizarea este metrica fundamentala de evaluare a unui model.

Exemplu din Lumea Reala

Un sistem de recunoastere a obiectelor in imagini primeste ca intrare vectori de caracteristici (pixeli, margini, texturi) si produce ca iesire etichete de clasa (masina, pieton, semafor). Generalizarea este critica: sistemul trebuie sa recunoasca obiecte in conditii noi de iluminare sau unghi.
