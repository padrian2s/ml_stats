Compromisul Fals Pozitiv vs Fals Negativ

Pagina analizeaza compromisul fundamental intre falsuri pozitive si falsuri negative in problemele de decizie binara, introducand matricea de pierdere asimetrica si pragul de decizie.

Fals pozitiv (False positive / Alarma falsa): Estimam y_hat=1 cand adevarul este y=0. Cost: L_FP.
Fals negativ (False negative / Detectie ratata): Estimam y_hat=0 cand adevarul este y=1. Cost: L_FN.
Eroarea de generalizare: L(θ, δ) = E_{(x,y)~p(x,y|θ)} [l(y, δ(x))] = ΣΣ L(y, δ(x))p(x,y|θ) (Ecuatia 5.109)
Pierderea asteptata posterioara: ρ(y_hat=0|x) = L_FN · p(y=1|x) (Ecuatia 5.111), ρ(y_hat=1|x) = L_FP · p(y=0|x) (Ecuatia 5.112)
Regula de decizie: Alegem y_hat=1 daca p(y=1|x)/p(y=0|x) > L_FP/L_FN (Ecuatia 5.114). Pragul τ = c/(1+c) unde c = L_FN/L_FP.

Comentariu

Rezumat

Pierderea 0-1 trateaza erorile simetric, dar in practica costurile sunt adesea asimetrice. Cu matricea de pierdere generala, regula de decizie devine: alegem clasa 1 daca raportul posterior depaseste raportul costurilor L_FP/L_FN. Daca falsul negativ costa de doua ori mai mult (c=2), pragul devine τ = 2/3, facand clasificatorul mai "agresiv" in detectia pozitivilor.

Tipare Cheie

Pierderea asteptata posterioara ρ(δ|D) = ∫ p(θ|D)L(θ,δ)dθ (Ecuatia 5.110) integreaza incertitudinea parametrilor in decizie. Aceasta difera de riscul frecventist (Ecuatia 6.47), care integreaza peste distributia de esantionare.

Aplicatii Practice

In medicina: detectarea cancerului are L_FN >> L_FP (o detectie ratata este mult mai grava decat o alarma falsa), deci pragul τ trebuie scazut. In filtrarea spam-ului: L_FP > L_FN (un email important marcat ca spam este mai grav decat spam-ul nedetectat).

Exemplu din Lumea Reala

Intr-un sistem de detectie a fraudei cu carduri: daca L_FN = 1000 EUR (frauda nedetectata) si L_FP = 5 EUR (inconvenient pentru client), atunci c = 200 si τ = 200/201 ≈ 0.995. Tranzactia este blocata doar daca probabilitatea de frauda depaseste 0.5%, ceea ce pare intuitiv.
