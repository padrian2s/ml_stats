Curbele ROC, Matricea de Confuzie si Metrici de Clasificare

Pagina introduce curbele ROC (Receiver Operating Characteristic), matricea de confuzie si metricile derivate: TPR (sensibilitate), FPR (rata falsilor pozitivi), AUC si EER.

Matricea de confuzie: Tabel care contine numarul de TP (adevarati pozitivi), FP (falsi pozitivi), FN (falsi negativi) si TN (adevarati negativi) rezultate din aplicarea regulii de decizie.
Curba ROC: Graficul TPR (sensibilitate) vs FPR (rata falsilor pozitivi) pe masura ce pragul τ variaza. Un clasificator perfect atinge coltul stanga-sus (FPR=0, TPR=1).
AUC (Area Under the Curve): Aria de sub curba ROC, sumarizeaza performanta intr-un singur numar. AUC=1 este perfect, AUC=0.5 este performanta aleatoare.
EER (Equal Error Rate): Punctul unde FPR = FNR = 1 - TPR. Se gaseste la intersectia curbei ROC cu diagonala din coltul dreapta-sus.
Metrici din matricea de confuzie: TPR = TP/N₊ = sensibilitate = recall; FPR = FP/N₋ = rata alarma falsa = eroare de tip I; TNR = TN/N₋ = specificitate.

Comentariu

Rezumat

Curbele ROC permit evaluarea unui clasificator fara a fixa un prag specific. Prin varierea pragului τ de la 0 la 1, se traseaza compromisul TPR vs FPR. AUC ofera un rezumat scalar: probabilitatea ca clasificatorul sa acorde un scor mai mare unui exemplu pozitiv decat unuia negativ. EER este alternativa care egaleaza cele doua tipuri de erori.

Tipare Cheie

Tabelele 5.2-5.4 sistematizeaza toate metricile derivabile din matricea de confuzie: p(y_hat|y) (TPR, FPR, TNR, FNR) si p(y|y_hat) (PPV/precizie, NPV, FDP). Distinctia intre aceste doua directii de conditionare este fundamentala.

Aplicatii Practice

Curbele ROC sunt standard in medicina (evaluarea testelor de diagnostic), securitate (sisteme biometrice - EER-ul este metrica principala) si invatarea automata (compararea clasificatorilor). AUC este preferat deoarece este invariant la pragul de decizie si la dezechilibrul claselor.

Exemplu din Lumea Reala

Un sistem de recunoastere faciala cu AUC=0.99 si EER=2% inseamna ca la punctul optim, 2% din utilizatorii legitimi sunt respinsi si 2% din impostori sunt acceptati. Curbele ROC permit alegerea pragului in functie de prioritatea securitatii vs. convenienta utilizatorilor.
