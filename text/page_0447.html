ICA: Minimizarea Informatiei Mutuale si Infomax

Pagina continua cu minimizarea multi-informatiei pentru ICA, principiul infomax (maximizarea informatiei mutuale) si demonstrarea echivalentei cu ML. Include si inceputul exercitiilor Capitolului 12.

Minimizarea multi-informatiei (Eq. 12.123): I(z) = suma H(z_j) - H(Vx) = suma H(z_j) - H(x) (cand V ortogonala). Se reduce la minimizarea suma H(z_j), echivalenta cu maximizarea negentropiei.
Principiul Infomax (Eq. 12.124): Maximizarea informatiei mutuale I(x; y) intr-o retea y_j = phi(v_j^T x) + epsilon. H(y) = suma E[log phi'(v_j^T x)] + log|det(V)|. Cu phi = sigmoid, infomax = ML cu distributie logistica.
Echivalenta fundamentala: Maximizarea negentropiei = Minimizarea multi-informatiei = Infomax = Maximum Likelihood. Toate aceste principii produc aceleasi estimari ale matricii de amestecare W.

Comentariu

Rezumat

Pagina completeaza demonstratia echivalentei tuturor principiilor de estimare ICA. Infomax, propus de Bell si Sejnowski (1995), maximizeaza informatia transmisa printr-o retea neuronala neliniara. Se demonstreaza ca aceasta este echivalenta cu ML cand neliniaritatile corespund CDF-urilor distributiilor surselor.

Tipare Cheie

Unificarea tuturor perspectivelor sub ML este un rezultat puternic: indiferent de motivatia intuitiva (independenta, non-Gaussianitate, informatie), formalismul matematic converge la aceeasi problema de optimizare.

Aplicatii Practice

Exercitiile capitolului acopera pasul M pentru FA, estimarea MAP pentru FA, euristica pentru utilitatea PCA si derivarea componentei principale secunde. Sunt exercitii fundamentale pentru intelegerea aprofundata.

Exemplu din Lumea Reala

In telecomunicatii, blind source separation (separarea oarba a surselor) foloseste principiul infomax pentru a recupera semnalele transmise din amestecuri receptionate, fara cunoasterea canalului de transmisie.
