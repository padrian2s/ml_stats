Sparse vector machines: phi(x) = [kappa(x, x_1), ..., kappa(x, x_N)]. Fiecare exemplu de antrenare este un centroid potential. Se foloseste regularizare de sparsitate (l1, group lasso, ARD) pentru a selecta automat un subset de centroizi. Alegeri: l1 -> LIVM, ARD/SBL -> RVM (Relevance Vector Machine).

Comentariu

Rezumat

Problema alegerii centroizilor este centrala pentru kernel machines. Sparse vector machines (LIVM, RVM) rezolva aceasta problema elegant: folosesc toate exemplele de antrenare ca centroizi potentiali, apoi selecteaza automat un subset prin regularizare sparse. Aceasta produce modele interpretabile (centroizii selectati sunt "relevanti" sau "de suport").

Tipare Cheie

Compromisul centroizi-bandwidth: cu putini centroizi, bandwidth-ul trebuie sa fie mare (acoperire globala). Cu multi centroizi, bandwidth-ul poate fi mic (detalii locale). Sparse vector machines automatizeaza acest compromis prin selectia data-driven a centroizilor relevanti.

Aplicatii Practice

RVM (Relevance Vector Machine) produce modele mai sparse decat SVM si ofera estimari probabilistice (predictive posterior), dar este mai lent la antrenare. LIVM cu LASSO este o alternativa mai rapida. In practica, SVM ramane cel mai popular datorita software-ului matur (libsvm, liblinear).

Exemplu din Lumea Reala

In detectia de anomalii in retele de calculatoare, RVM selecteaza automat un set mic de pattern-uri "relevante" din traficul normal, producand un model compact care poate fi deployat pe routere cu memorie limitata.
