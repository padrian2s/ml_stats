Detalii despre single link, complete link si average link

Aceasta pagina detaliaza matematic cele trei tipuri de legatura in clustering-ul aglomerativ, cu proprietatile si compromisurile fiecareia.

Single link: d_SL(G,H) = min_{i in G, i' in H} d_{i,i'}. Arborele rezultat este un arbore de acoperire minima (minimum spanning tree). Se poate implementa in O(N^2), mai rapid decat celelalte variante O(N^3).
Complete link: d_CL(G,H) = max_{i in G, i' in H} d_{i,i'}. Produce clustere compacte cu diametru mic, dar necesita ca toate perechile de puncte sa fie apropiate pentru a imbina grupuri.
Proprietatea de compactitate: Toate observatiile dintr-un cluster ar trebui sa fie similare intre ele. Single linkage o violeaza (poate produce clustere cu diametru mare), complete linkage o impune strict.
Average link ca compromis: d_avg(G,H) = (1/n_G*n_H) * suma suma d_{i,i'}, produce clustere relativ compacte si bine separate. Dezavantaj: nu este invarianta la transformari monotone ale distantelor, spre deosebire de single si complete linkage.

Comentariu

Rezumat

Pagina formuleaza matematic fiecare tip de legatura si analizeaza compromisurile. Single link este rapida (O(N^2)) si echivalenta cu arborele de acoperire minima, dar produce efect de lant. Complete link produce clustere compacte dar poate fi prea conservatoare. Average link este un compromis bun dar sensibila la scala de masurare.

Tipare Cheie

Conexiunea single linkage - arbore de acoperire minima este o legatura eleganta intre clustering si teoria grafurilor. Invarianta la transformari monotone (single si complete) vs. sensibilitatea la scala (average) este o proprietate practica importanta in alegerea metodei.

Aplicatii Practice

In analiza filogenetica, UPGMA (un tip de average linkage) presupune un ceas molecular constant si produce arbori ultrametrici. Neighbor-joining, o alternativa mai flexibila, nu face aceasta presupunere si este preferata pentru arbori filogenetici precisi.

Exemplu din Lumea Reala

In detectia fraudelor in tranzactii financiare, single linkage poate identifica "lanturi" de tranzactii suspecte conectate prin intermediari, in timp ce complete linkage ar identifica doar grupuri compacte de tranzactii foarte similare.
